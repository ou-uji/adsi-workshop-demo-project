# 打刻メモ（備考）入力 — 確定仕様（ゲート①）

> 元 Q&A: [新規機能_打刻メモ_QA.md](./新規機能_打刻メモ_QA.md)
> ブランチ: `feature/attendance-memo`

## 概要

打刻（出勤・退勤）時に、任意で備考メモを入力・保存できるようにする。
保存したメモは勤怠履歴画面で確認でき、本人はあとから編集・削除できる。

例:「電車遅延のため遅刻」「客先直行」

## ユーザーストーリー

- 社員として、打刻時に事情をメモとして残したい（遅刻理由・直行直帰など）
- 社員として、あとから自分のメモを修正・削除したい
- 上長・管理者として、部下/全社員の打刻メモを勤怠履歴で確認したい

## 確定仕様

| # | 項目 | 決定 |
|---|------|------|
| 1 | 入力必須？ | **任意**（空欄 OK） |
| 2 | 文字数上限 | **200 文字**（全角/半角区別なし、コードポイント数） |
| 3 | 対象 | **出勤・退勤の両方**にそれぞれ付けられる |
| 4 | 編集・削除 | **可能**。**承認不要**、本人がその場で自由に |
| 5 | 閲覧範囲 | **本人・上長・管理者**（既存の勤怠履歴の閲覧範囲に準拠） |
| 6 | 編集・削除できる人 | **本人のみ**。上長・管理者は閲覧のみ |
| 7 | 操作場所 | **勤怠履歴画面（月別テーブル）**の各行 |
| 8 | DB 設計 | **`clock_in_memo` / `clock_out_memo` の 2 カラム**に分離 |

## 受け入れ基準（Acceptance Criteria）

### 打刻時の保存
- [ ] 出勤打刻時にメモを入力すると、`clock_in_memo` に保存される
- [ ] 退勤打刻時にメモを入力すると、`clock_out_memo` に保存される
- [ ] メモ未入力（空欄）でも打刻は正常に完了する（null で保存）
- [ ] 200 文字を超えるメモは 400 エラー（バリデーションエラー）になる
- [ ] 200 文字ちょうどは保存できる

### 履歴表示
- [ ] 勤怠履歴画面の各行に出勤メモ・退勤メモが表示される
- [ ] メモが無い日は空欄で表示される
- [ ] 上長・管理者も部下/全社員のメモを閲覧できる

### 編集・削除
- [ ] 本人は履歴画面の各行から自分のメモを編集できる
- [ ] 本人は自分のメモを削除（空に）できる
- [ ] 編集・削除に上長承認は不要（即時反映）
- [ ] 他人のメモは編集・削除できない（本人のみ / 403）

## スコープ外

- メモの変更履歴・監査ログ（誰がいつ変えたか）は持たない
- メモに対する承認フロー（勤怠時刻修正の Unit 05 とは独立）
- リッチテキスト・添付ファイル・定型文テンプレート

## 影響範囲（設計フェーズで詳細化）

- **DB**: `V5` マイグレーション（`attendance_records` に 2 カラム追加）
- **Backend**: Entity / DTO / Controller（打刻 API に body 追加 + メモ更新 API 新設）/ Service
- **Frontend**: 打刻画面（ClockButtons にメモ入力）/ 勤怠履歴（メモ列 + 編集 UI）/ API 層
