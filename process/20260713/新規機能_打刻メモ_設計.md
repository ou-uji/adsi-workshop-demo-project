# 打刻メモ（備考）入力 — 実装設計・Plan（ゲート②）

> 仕様（ゲート①）: [新規機能_打刻メモ_仕様(SDD).md](./新規機能_打刻メモ_仕様(SDD).md)
> 元 Q&A: [新規機能_打刻メモ_QA.md](./新規機能_打刻メモ_QA.md)
> ブランチ: `feature/attendance-memo`

## Context

打刻（出勤・退勤）時に任意の備考メモを残せるようにする機能追加。
現状 `AttendanceRecord` は memo フィールドを一切持たず、打刻 API は `employeeId`（クエリ）のみで body なし。
ここに出勤メモ・退勤メモを追加し、勤怠履歴画面で閲覧＋本人がその場で編集・削除できるようにする。

**確定仕様**: 任意 / 200文字 / 出退勤両方 / 承認不要・本人がその場で編集削除 / 閲覧は本人＋上長＋管理者 /
編集は本人のみ / 履歴画面の行から操作 / DB は `clock_in_memo`・`clock_out_memo` の2カラム。

## 設計判断（既存パターン準拠）

「新しい書き方を発明しない — 既存の流儀に乗る」方針で、既存コードのパターンに合わせた:

- 打刻時のメモ送信は **クエリ `employeeId` を維持しつつ JSON body で `{memo}` を送る**
  （`createCorrection` の「クエリで id + JSON body」先例に準拠）
- メモ編集は **1レコードの clock_in_memo/clock_out_memo を1回の PATCH でまとめて更新**（削除は空文字→null）
- 認可は **サービス層で `record.getEmployee().getId()` と acting employeeId を比較**
  （`CorrectionServiceImpl.validateApprover` パターン。method-level `@PreAuthorize` は本 repo で未使用）
- 楽観ロックは既存の **`@Version` + 手動 `validateVersion`（409）** に合わせる
- フロントは **zod/react-hook-form なし**・native validation・生 `<textarea>`・`FormDialog` 再利用・
  楽観更新なし（invalidate のみ）

## Backend

### 1. Flyway migration（新規）
`packages/backend/src/main/resources/db/migration/V5__add_memo_to_attendance_records.sql`
```sql
ALTER TABLE attendance_records ADD COLUMN clock_in_memo  VARCHAR(200);
ALTER TABLE attendance_records ADD COLUMN clock_out_memo VARCHAR(200);
```
- nullable、`VARCHAR(200)`（`reason` の `VARCHAR(500)` に倣うが仕様上限 200）。`ddl-auto` 禁止なので必須。
- 最新は V4。次は V5。

### 2. Entity
`.../attendance/entity/AttendanceRecord.java`
- `@Column(length = 200) private String clockInMemo;` / `clockOutMemo;` を追加（既存 `reason` の `@Column(length=...)` スタイル）。

### 3. DTO
- `.../attendance/dto/AttendanceRecordResponse.java`: record に `String clockInMemo, String clockOutMemo` を追加、
  `from()` にマッピング追加。**`Long version` も追加**（編集時にクライアントへ返す必要 — correction 先例）。
- 新規 `.../attendance/dto/ClockRequest.java`: `record ClockRequest(@Size(max = 200) String memo) {}`
  （memo 任意なので `@NotNull` は付けない）。
- 新規 `.../attendance/dto/MemoUpdateRequest.java`:
  `record MemoUpdateRequest(@Size(max = 200) String clockInMemo, @Size(max = 200) String clockOutMemo, @NotNull Long version) {}`

### 4. Repository
変更なし（既存の `findByEmployeeIdAndWorkDateAndClockOutIsNull` 等で足りる。編集用 `findById` は JpaRepository 標準）。

### 5. Service
`.../attendance/service/AttendanceService.java`（interface）+ `AttendanceServiceImpl.java`
- `clockIn(UUID employeeId, String memo)` / `clockOut(UUID employeeId, String memo)` にシグネチャ拡張。
  ビルダーに `.clockInMemo(memo)` / clockOut は `record.setClockOutMemo(memo)`。空文字は null 化（`blankToNull` ヘルパ）。
- 新規 `updateMemo(UUID recordId, UUID employeeId, String clockInMemo, String clockOutMemo, Long version)`:
  1. `findById(recordId).orElseThrow(EntityNotFoundException)`
  2. **所有者チェック**: `record.getEmployee().getId().equals(employeeId)` でなければ `ResponseStatusException(FORBIDDEN)`
  3. `validateVersion(record, version)`（correction と同じ 409 ロジックを attendance にも用意）
  4. `setClockInMemo`/`setClockOutMemo`（空文字→null）→ `save` → `AttendanceRecordResponse.from`
- `@Transactional`（write）を付与。時刻は既存どおり注入された `Clock` を使用。

### 6. Controller
`.../attendance/controller/AttendanceController.java`
- `clockIn`: `@RequestParam UUID employeeId, @Valid @RequestBody(required = false) ClockRequest request` に変更。
  `request == null ? null : request.memo()` を service へ。**後方互換**（body なしでも 201 で通る）。
- `clockOut`: 同様。
- 新規 `@PatchMapping("/records/{id}/memo")`:
  `updateMemo(@PathVariable UUID id, @RequestParam UUID employeeId, @Valid @RequestBody MemoUpdateRequest request)` → 200。

### 7. SecurityConfig
`.../common/config/SecurityConfig.java` の `configureAuthorization`
- PATCH `/api/attendance/records/*/memo` を `.authenticated()` で許可（所有者チェックはサービス層）。
  既存の `/api/attendance/**` が `.authenticated()` で括られていれば追加不要 → 実装時に確認。

## Frontend

### 8. 型・API 層
`.../features/attendance/attendance-api.ts`
- `AttendanceRecordResponse` に `clockInMemo: string | null; clockOutMemo: string | null; version: number` を追加。
- `clockIn(employeeId, memo?)` / `clockOut(employeeId, memo?)`: 第2引数 memo を受け、`apiClient.post(url, { memo })`
  （body 追加。url のクエリは維持）。
- 新規 `updateMemo(recordId, employeeId, body)`:
  `apiClient.patch(\`/api/attendance/records/${recordId}/memo?employeeId=${employeeId}\`, body)`。
  apiClient に patch が無ければ確認・追加。

### 9. Hook
`.../features/attendance/useAttendance.ts`
- `useClockIn`/`useClockOut`: `mutationFn: (memo?: string) => clockIn(user!.id, memo)` に変更。
  **`onError` toast を追加**（既存は未実装、correction 先例に合わせる）。invalidate は `TODAY_STATUS_KEY`。
- 新規 `useUpdateMemo(month)`: `useMutation`、成功で `HISTORY_KEY` を invalidate + `toast.success("メモを更新しました")`、
  `onError` で `ApiClientError.detail`。

### 10. 打刻画面 UI
`.../features/attendance/ClockButtons.tsx`
- 出勤メモ・退勤メモ用に `useState`（もしくは打刻直前に入力欄）。生 `<textarea maxLength={200} rows={2}>`
  （CorrectionForm のスタイル文字列を流用）+ `<Label>`。
- `clockInMutation.mutate(memo)` / clockOut 同様。打刻成功後に入力欄をクリア。空欄OK。

### 11. 履歴画面 — 表示＋編集
- `.../features/attendance/AttendanceTable.tsx`: 「備考」列を追加（出勤メモ/退勤メモを表示、無ければ空）。
  本人の行にのみ編集ボタン（鉛筆）を出す。
  ※ 履歴は本人ページ（`/history`）＝常に本人なので全行編集可。上長/管理者の閲覧ページ（team/all）は**編集ボタン非表示**。
- 新規 `.../features/attendance/MemoEditDialog.tsx`: `FormDialog` を再利用。
  clock_in_memo/clock_out_memo の2 textarea + version を保持。
  保存で `useUpdateMemo().mutate(..., { onSuccess: () => setOpen(false) })`、削除は空にして保存。
- `.../app/(authenticated)/history/page.tsx`: ダイアログ open state と選択レコードを保持し `AttendanceTable` に渡す。
