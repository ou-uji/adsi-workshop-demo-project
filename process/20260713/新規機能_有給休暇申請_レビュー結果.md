## test-reviewer の HIGH 指摘と対応（実コードで裏取り）

subagent の findings を鵜呑みにせず、実際のテストファイルを確認して**正当性を検証**した。

### HIGH① 理由500文字ちょうど成功 + 501文字→400 の境界テストが無い（裏取り: **正当**）

- 裏取り: `LeaveRequestControllerTest` を確認 → `blank`・`missingTargetDate` はあるが maxLength 境界テストが実在せず。テスト仕様書には C3（501→400）を記載していたが実装漏れ。
- **対応**: `create_tooLongReason_returns400`（501字→400）+ `create_maxLengthReason_returns201`（500字ちょうど→201）を追加。

### HIGH② 他人の申請が自分の一覧に出ない（isolation）が未証明（裏取り: **正当**）

- 裏取り: `LeaveRequestRepositoryTest` R1 は同一 requester のデータのみで、他 requester の除外を証明していなかった。
- **対応**: setUp に別部署（Sales）の別社員 `otherEmployee` を追加。R1 を「自分2件 + 他人1件 → 自分の2件のみ」の isolation 証明に強化。

### MEDIUM 系の対応

- **他部署が承認待ちに出ない + PENDING 以外除外**（R3 強化）: 他部署 PENDING + 自部署 APPROVED を混ぜ、「自部署 PENDING 1件のみ」を二重に証明。
- **reject の version 409（対称性）**: `reject_versionMismatch_throwsConflict` を追加（approve だけでなく reject も楽観ロックが効くことを証明）。
